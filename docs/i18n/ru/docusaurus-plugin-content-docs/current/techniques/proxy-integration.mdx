При настройке прокси в логах будут отображаться соответствующие сообщения:

Could you please provide the text you'd like me to translate? Once you share the English content, I'll be happy to translate it to Russian for you.

NestCord поддерживает конфигурацию прокси для всех подключений к Discord API. Это полезно при развертывании в корпоративных сетях или при использовании VPN.

### proxyPath
- **Type**: `строка`
- **Description**: Полный URL прокси-сервера
- **Examples**: 
  - `'http://proxy.company.com:8080'`
  - `'https://user:pass@proxy.company.com:8080'`
  - `'socks5://proxy.company.com:1080'`

1. **Переменные окружения**: При инициализации модуля переменные `HTTP_PROXY` и `HTTPS_PROXY` устанавливаются автоматически
2. **Discord.js Client**: Все подключения к Discord API будут осуществляться через указанный прокси
3. **NestCordService**: REST API запросы также будут использовать настройки прокси

```typescript title="src/app.module.ts"
import { NestCordModule } from '@globalart/nestcord';
import { GatewayIntentBits } from 'discord.js';

@Module({
  imports: [
    NestCordModule.forRoot({
      token: process.env.DISCORD_TOKEN,
      intents: [GatewayIntentBits.Guilds],
      proxyPath: 'http://proxy.company.com:8080',
    }),
  ],
})
export class AppModule {}
```

#### NestCordProxyOptions

**NestCordProxyOptions** - это интерфейс конфигурации для настройки прокси-параметров в приложении NestCord.

**Основные свойства:**

- `enabled` (boolean) - включение или отключение прокси
- `host` (string) - адрес хоста прокси-сервера
- `port` (number) - порт прокси-сервера
- `protocol` (string) - протокол прокси (http, https, socks5)
- `auth` (object) - учетные данные для аутентификации:
  - `username` (string) - имя пользователя
  - `password` (string) - пароль
- `rejectUnauthorized` (boolean) - проверка SSL-сертификатов
- `timeout` (number) - время ожидания соединения в миллисекундах

**Пример использования:**

```typescript
const proxyOptions: NestCordProxyOptions = {
  enabled: true,
  host: 'proxy.example.com',
  port: 8080,
  protocol: 'http',
  auth: {
    username: 'user',
    password: 'pass'
  },
  rejectUnauthorized: false,
  timeout: 5000
};
```

- Параметры прокси применяются глобально для всего приложения
- Поддерживаются прокси HTTP, HTTPS и SOCKS5
- Если конфигурация прокси не удаётся, приложение продолжит работу без прокси
- Параметры прокси влияют на все соединения Discord.js, включая WebSocket и REST API

```typescript title="src/app.module.ts"
@Module({
  imports: [
    NestCordModule.forRoot({
      token: process.env.DISCORD_TOKEN,
      intents: [GatewayIntentBits.Guilds],
      proxy: {
        host: 'proxy.company.com',
        port: 8080,
        protocol: 'http',
        auth: {
          username: 'user',
          password: 'password',
        },
      },
    }),
  ],
})
export class AppModule {}
```

### Прокси с аутентификацией
```typescript title="src/app.module.ts"
NestCordModule.forRoot({
  token: process.env.DISCORD_TOKEN,
  intents: [GatewayIntentBits.Guilds],
  proxy: {
    host: 'secure-proxy.company.com',
    port: 8080,
    protocol: 'https',
    auth: {
      username: process.env.PROXY_USER,
      password: process.env.PROXY_PASS,
    },
  },
})
```

```typescript title="src/my.service.ts"
import { Injectable } from '@nestjs/common';
import { NestCordService } from '@globalart/nestcord';

@Injectable()
export class MyService {
  constructor(private readonly nestcord: NestCordService) {}

  getProxyInfo() {
    const proxyConfig = this.nestcord.getProxyConfig();
    console.log('Текущая конфигурация прокси:', proxyConfig);
  }
}
```
