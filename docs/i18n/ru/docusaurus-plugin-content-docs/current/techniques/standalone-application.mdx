Для этого измените функцию bootstrap в файле `main.ts` вашего проекта на что-то вроде этого:

Если вы инициализировали своё приложение с помощью Nest CLI, фреймворк Express будет установлен по умолчанию вместе с Nest. Nest и NestCord не требуют Express для работы. Поэтому если вам не нужен веб-сервер, вы можете удалить Express.

```bash
npm un @nestjs/platform-express @types/express
```

```typescript title="src/main.ts"
import { NestFactory } from '@nestjs/core';

async function bootstrap() {
    const app = await NestFactory.createApplicationContext(AppModule);
}

bootstrap();
```

:::caution
Но когда вы запускаете автономное приложение, вы не можете использовать глобальные усилители, потому что они не зарегистрированы в контексте приложения. Пока [проблема](https://github.com/nestjs/nest/issues/9017) не решена, вы можете использовать локальные усилители или запустить приложение через метод `app.init()`.
:::

Это инициализирует Nest как **автономное приложение** (без каких-либо сетевых слушателей).
