title: Автономное приложение
sidebar_position: 3
---

Если вы инициализировали ваше приложение с помощью Nest CLI, Express Framework будет установлен по умолчанию вместе с Nest. Nest и NestCord не требуют Express для работы. Так что если вам не нужен веб-сервер, вы можете удалить Экспресс.

Чтобы сделать это, измените функцию bootstrap в файле `main.ts` вашего проекта на что-то вроде следующего:

```typescript title="src/main.ts"
импорт { NestFactory } из '@nestjs/core';

async function bootstrap() {
    const app = await NestFactory.createApplicationContext(AppModule);
}

bootstrap();
```

Это инициализирует Nest как **автономное приложение** (без каких-либо слушателей сети).

Все что осталось - это удалить неиспользуемые зависимости:

```bash
npm un @nestjs/platform-express @types/express
```

:::caution
Но когда вы запускаете автономное приложение, вы не можете использовать глобальные усилители из-за того, что они не зарегистрированы в контексте приложения. В то время как [issue](https://github.com/nestjs/nest/issues/9017) не решено, вместо этого вы можете использовать локальные усилители или запускать приложение методом `app.init()`.
:::
