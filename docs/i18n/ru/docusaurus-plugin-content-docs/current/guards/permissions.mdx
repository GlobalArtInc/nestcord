```typescript title="src/filters/permissions.filter.ts"
import { ArgumentsHost, Catch, ExceptionFilter } from '@nestjs/common';
import { NestCordArgumentsHost, InsufficientPermissionsException, AnyContext } from '@globalart/nestcord';

@Catch(InsufficientPermissionsException)
export class PermissionsExceptionFilter implements ExceptionFilter {
  catch(exception: InsufficientPermissionsException, host: ArgumentsHost) {
    const [interaction] = NestCordArgumentsHost.create(host).getContext<AnyContext>();
    return interaction.reply({
      content: '🚫 У вас нет разрешения на использование этой команды.',
      ephemeral: true,
    });
  }
}
```

## Как это работает

`@Permissions()` проверяет **права участника** только (что пользователь может делать на сервере). Это **не** проверяет права бота — добавьте отдельный guard для этого, если требуется.

:::info ДМ
В прямых сообщениях `memberPermissions` имеет значение `null`, поэтому любая команда, украшенная `@Permissions()`, будет заблокирована в ДМ.
:::

```typescript
@Permissions(PermissionFlagsBits.ManageMessages, PermissionFlagsBits.ViewChannel)
@SlashCommand({ name: 'clear', description: 'Удалить сообщения' })
async onClear(@Context() [interaction]: SlashCommandContext) { ... }
```

Зарегистрируйте фильтр на обработчик, на класс или глобально:

```typescript
@Permissions(PermissionFlagsBits.Administrator)
@Injectable()
export class AdminCommands {
  @SlashCommand({ name: 'setup', description: 'Начальная настройка сервера' })
  async onSetup(@Context() [interaction]: SlashCommandContext) { ... }

  @SlashCommand({ name: 'reset', description: 'Сброс конфигурации' })
  async onReset(@Context() [interaction]: SlashCommandContext) { ... }
}
```

Примените `@Permissions()` к любому обработчику слеш-команды, контекстного меню, кнопки, выпадающего меню или текстовой команды. Передайте одно или несколько значений [`PermissionFlagsBits`](https://discord.js.org/docs/packages/discord.js/main/PermissionFlagsBits:Variable) — участник должен иметь **все** из них.

```typescript
// По обработчику
@UseFilters(PermissionsExceptionFilter)
@Permissions(PermissionFlagsBits.BanMembers)
@SlashCommand({ name: 'ban', description: 'Заблокировать участника' })
async onBan(...) { ... }

// По классу
@UseFilters(PermissionsExceptionFilter)
@Injectable()
export class ModerationCommands { ... }

// Глобально (main.ts)
app.useGlobalFilters(new PermissionsExceptionFilter());
```

```typescript title="src/moderation.commands.ts"
import { Injectable } from '@nestjs/common';
import { PermissionFlagsBits } from 'discord.js';
import { Context, Permissions, SlashCommand, SlashCommandContext } from '@globalart/nestcord';

@Injectable()
export class ModerationCommands {
  @Permissions(PermissionFlagsBits.KickMembers)
  @SlashCommand({ name: 'kick', description: 'Исключить участника' })
  async onKick(@Context() [interaction]: SlashCommandContext) {
    // до этой точки доступны только члены с разрешением KickMembers
  }
}
```

Декоратор `@Permissions()` ограничивает выполнение команды участниками Discord, обладающими указанными разрешениями. Он работает совместно с `PermissionsGuard`, который проверяет `interaction.memberPermissions` (или `message.member.permissions` для текстовых команд) перед вызовом обработчика.

```
Пользователь вызывает /kick
        │
        ▼
 PermissionsGuard
        │
  есть KickMembers?
   ┌────┴────┐
  да        нет
   │         │
   ▼         ▼
handler   выбрасывает InsufficientPermissionsException
             │
             ▼
    PermissionsExceptionFilter
             │
             ▼
    временный ответ пользователю
```
