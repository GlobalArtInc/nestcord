## Git Commit Message Convention

:::info

Это адаптируется к [конвенции коммита Ангела](https://github.com/conventional-changelog/conventional-changelog/tree/master/packages/conventional-changelog-angular).

:::

#### TL;DR:

Заголовки коммитов должны совпадать со следующим выражением:

```js
/^(revert: )?(feat|fix|docs|style|refactor|perf|test|workflow|build|ci|chore|types|wip)(\(.+\))?: .{1,72}/;
```

### Полный формат сообщений

Сообщение о коммите состоит из **заголовка**, **body** и **footer**. Заголовок имеет тип **, **scope** и **subject\*\*:

```
<type>(<scope>): <subject>
<BLANK LINE>
<body>
<BLANK LINE>
<footer>
```

**Заголовок** является обязательным, и **область заголовка** заголовка является необязательной.

### Восстановить

Если коммит вернет предыдущее коммит, он должен начинаться с «revert:», после чего следует заголовок отмененного коммита. In the body, it should say: `This reverts commit <hash>.`, where the hash is the SHA of the commit being reverted.

### Тип

Если префикс `feat`, `fix` или `perf`, он появится в списке изменений. Тем не менее, если есть [ОБРЯЗАТЬ ИЗМЕНЕНИЕ](#footer), коммит всегда будет отображаться в списке изменений.

Другие префиксы зависят от вашего усмотрения. Предложенные префиксы: `docs`, `chore`, `style`, `refactor` и `test` для задач, не связанных с changelog.

### Сфера охвата

Область действия ссылается на какой раздел приложения в котором были внесены изменения, такие как `SlashCommand`, `ContextMenu`, или `MessageComponents`

### Тема

Тема содержит краткое описание изменения:

- Используйте императив, настоящее десять: «изменить», не «изменился» или «изменения»
- не заглавьте первую букву
- не завершать ваше сообщение через период (.)

### Тело

Like in the **subject**, use the imperative, present tense
The body should include the motivation for the change and difference with the previous behavior.

### Колонтитул

Нижний колонтитул должен содержать любую информацию о **Нарушение изменений**, а также место для
ссылки на GitHub вопросы, которые это фиксирует **Закрытия**.

**Переменные изменения** должны начинаться со слова `BREAKING CHANGE:` с пробелом или двумя новыми строками. Оставшиеся сообщения о коммите используются для этого.
